# Deploy

## Description

### Сценарии

- Регистрация по почте и паролю
- Аутентификация:
  - через jwt access токен
  - по почте и паролю,
    если refresh токен истёк
- Получение refresh и access токенов,
    с помощью старого refresh токена
- Выход из аккаунта, очистка access и refresh токенов
- Получение пагинированных данных о пользователях
  - сначала получаем количество страниц
  - потом делаем пагинированный запрос
    (page = 0 [start = 0, step = 10])
- Пагинированный поиск по пользователям
    (запрос с ilike по имени и фамилии)

### Хранение

В postgres храняться данные о пользователе
    (почта, пароль, имя, фамилия и тд)

В redis храняться выданные access,
    refresh токены и некоторые метаданные
